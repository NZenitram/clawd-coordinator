# Clawd Coordinator

CLI tool for orchestrating remote Claude Code sessions across machines via WebSocket. Lets you dispatch prompts to Claude Code instances running on cloud VMs, stream results back in real time, and fan out work across multiple machines — all from your local terminal or from within a Claude Code session via bash.

## Architecture

Three components, one npm package, three modes via subcommand:

```
Local Machine                          Remote Machines
┌─────────────────────┐
│  Claude Code ──bash──┤
│                      │                ┌──────────────────┐
│  CLI (coord) ────────┤──── WSS ──────│  Remote Agent A   │
│                      │                └──────────────────┘
│  Coordinator         │                ┌──────────────────┐
│  (WebSocket server)  │──── WSS ──────│  Remote Agent B   │
│                      │                └──────────────────┘
│  Tailscale Funnel    │                ┌──────────────────┐
│  (exposes port 8080) │──── WSS ──────│  Remote Agent C   │
└─────────────────────┘                └──────────────────┘
```

### Coordinator (`coord serve`)

WebSocket server running locally. Accepts connections on two paths:
- `/agent?token=` — remote agent daemons
- `/cli?token=` — CLI clients

Manages agent registry, task dispatch, output streaming to subscribed CLI connections. All state is in-memory (V1).

### Remote Agent (`coord agent`)

Lightweight daemon on each remote machine. Connects outbound to the coordinator (no inbound ports needed). On task receipt, spawns `claude -p --verbose --output-format stream-json` as a child process and streams output back over WebSocket. Auto-reconnects with exponential backoff.

Prerequisites on remote machine: Claude Code installed and authenticated, Node.js.

### CLI (`coord run`, `coord agents`, etc.)

Connects to the coordinator over the same WebSocket protocol. Dispatches tasks, streams results, lists agents and tasks.

## Protocol

All communication uses JSON messages over WebSocket with a shared envelope:
```typescript
{ id: string, type: string, timestamp: number, payload: T }
```

Message types: `agent:register`, `agent:heartbeat`, `task:dispatch`, `task:output`, `task:complete`, `task:error`, `cli:request`, `cli:response`

Defined in `src/protocol/messages.ts`.

## Auth

Single shared token for V1, generated by `coord init`, stored at `~/.coord/config.json`. Agents and CLI both present it as a query parameter on WebSocket connect.

## CLI Commands

| Command | Description |
|---------|-------------|
| `coord init` | Generate config and auth token |
| `coord serve` | Start coordinator WebSocket server |
| `coord agent --url --token --name` | Start remote agent daemon |
| `coord agents` | List connected agents |
| `coord run "prompt" --on <agent>` | Dispatch prompt, stream output |
| `coord run "prompt" --on <agent> --bg` | Dispatch and background |
| `coord fan-out "prompt" --on <a,b,c>` | Dispatch to multiple agents |
| `coord tasks` | List tasks |
| `coord attach <task-id>` | Stream output from running task |
| `coord result <task-id>` | Get completed task output |
| `coord sessions --on <agent>` | List sessions (stub) |
| `coord resume <session-id> --on <agent>` | Resume session (stub) |

## V1 Limitations

- In-memory state only — task history lost on coordinator restart
- Single-user, single shared token
- No task queuing — busy agents reject new tasks immediately
- No persistent sessions listing (sessions/resume are stubs)
- Coordinator must be reachable by agents (Tailscale Funnel for local use)

## Security Notes

- Spawned `claude` processes on remote agents inherit the full process environment. Operators should minimize sensitive env vars on agent machines or run agents in containers with restricted environments.
- Auth uses a single shared token (V1). All agents and CLI clients share the same token. Agent names are self-reported.
- Config file at `~/.coord/config.json` contains the auth token. File permissions are set to 0600 on creation.
- The coordinator should be run behind TLS (use `--tls-cert`/`--tls-key` or Tailscale Funnel).
