## Phase A: Known Gaps (9 hours)

Quick fixes from review findings. All independent, can be done in any order.

### A1: Worktree Orphan Cleanup on SIGKILL (~2h)

Add `pruneOrphans(baseDir)` to `WorktreeStrategy` in `src/agent/isolation.ts`. Runs `git worktree list --porcelain`, filters to paths under `.worktrees/`, removes stale ones with `git worktree remove --force`. Called from `AgentDaemon.start()` when `isolation === 'worktree'`, before connecting.

**Files:** `src/agent/isolation.ts`, `src/agent/daemon.ts`, `tests/agent/isolation.test.ts`

### A2: TLS Shutdown Socket Drain (~1.5h)

Track TLS sockets in `httpsConnections` set (same pattern as `httpConnections`). Destroy all on `stop()` before calling `httpsServer.close()`.

**Files:** `src/coordinator/server.ts`, `tests/coordinator/server.test.ts`

### A3: WebSocket Backpressure (~2.5h)

Create `safeSend(ws, data, highWaterMark)` in `src/shared/ws-utils.ts`. Checks `ws.bufferedAmount >= highWaterMark` (1MB default). For task output: drop and log warning. For control messages (complete/error): always send. Apply in server output fan-out and daemon onOutput callback.

**Files:** New `src/shared/ws-utils.ts`, `src/coordinator/server.ts`, `src/agent/daemon.ts`, `tests/shared/ws-utils.test.ts`

### A4: Rate Limiting (~3h)

Sliding-window token bucket in `src/shared/rate-limiter.ts`. Two limits:
- Per-IP connection count (default 10) — enforced in `verifyClient`
- Per-socket message rate (default 100/s for CLI, 500/s for agents) — checked in message handlers, close with 4008 if exceeded

**Files:** New `src/shared/rate-limiter.ts`, `src/coordinator/server.ts`, `tests/shared/rate-limiter.test.ts`

## Phase B: Task Resilience (14-18 hours)

### B1: Retry / Dead Letter Handling (6-8h)

**Data model:** Add to `Task` interface: `retryCount`, `maxRetries`, `retryPolicy`, `deadLettered`, `lastError`. Add `'dead-letter'` to `TaskStatus` union.

**Retry policy:** `{ maxRetries: number, baseDelayMs: number, maxDelayMs: number, retryableErrors: string[] }`. Default: 3 retries, 5s base, 60s max.

**Error classification:** Retryable = agent disconnect, timeout, socket not open, capacity. Permanent = claude exit code errors, invalid input.

**Logic:** In `task:error` handler and agent disconnect path: check retryable + under max retries -> increment count, set status back to pending, schedule re-enqueue via `setTimeout` with exponential backoff. Otherwise -> set `deadLettered: true`, status `dead-letter`.

**New CLI command:** `coord dead-letter` lists dead-lettered tasks.

**Files:** `src/coordinator/tasks.ts`, `src/coordinator/server.ts`, `src/protocol/messages.ts` (add `retryAttempt` to dispatch payload), `src/coordinator/rest.ts` (dead-letter filter)

### B2: Cross-Session Agent Communication (8-10h)

**Protocol messages:**
- `agent:message` — `{ fromAgent, toAgent, correlationId, topic, body }`
- `agent:message-reply` — `{ fromAgent, toAgent, correlationId, body }`
- `agent:message-ack` — `{ correlationId, status: 'delivered' | 'agent-offline' | 'unknown-agent' }`

**Coordinator:** Relay messages between agents via `agentSockets` lookup. Send ack to sender.

**Agent:** Surface incoming messages as task output annotations. Provide reply mechanism.

**REST:** Optional `POST /api/message` for CLI-initiated messages.

**Files:** `src/protocol/messages.ts`, `src/coordinator/server.ts`, `src/agent/daemon.ts`, `src/coordinator/rest.ts`

## Phase D: User Management (20-25 hours)

**Data model:**
```sql
CREATE TABLE users (id, username UNIQUE, role DEFAULT 'operator', created_at);
CREATE TABLE api_keys (id, user_id REFERENCES users, key_hash, label, created_at, revoked_at);
```
Add `owner_user_id` to tasks table.

**Roles:** admin (all ops + user management), operator (dispatch + own tasks + list agents), viewer (read-only).

**Auth flow:** `resolveUser(token, store)` hashes token, looks up in api_keys, returns user+role. Falls back to legacy shared token (treated as admin).

**REST endpoints:** `POST /api/users`, `POST /api/users/:id/keys`, `DELETE /api/keys/:id`.

**Files:** New `src/coordinator/user-store.ts`, `src/coordinator/rbac.ts`. Modify `src/shared/auth.ts`, `src/coordinator/server.ts`, `src/coordinator/rest.ts`, `src/coordinator/sqlite-store.ts`, `src/coordinator/tasks.ts`

## Phase E: Hosted Relay (35-45 hours)

**Requires Phase D complete.**

**Data model:**
```sql
CREATE TABLE orgs (id, name UNIQUE, created_at);
CREATE TABLE org_members (org_id REFERENCES orgs, user_id REFERENCES users, role, PRIMARY KEY(org_id, user_id));
```
Add `org_id` to tasks. Persistent agent registration per org.

**Architecture:** Coordinator becomes multi-tenant. All state maps (`agentSockets`, `cliSockets`, `taskSubscribers`, `taskOwners`) scoped by org ID. Connection routing: `/agent?org=<orgId>`, `/cli?org=<orgId>`.

**Registration:** `POST /api/orgs` (create org), `POST /api/orgs/:id/invite`, `POST /api/register` (self-service signup).

**Risks:** sql.js may bottleneck under multi-tenant load (consider PostgreSQL migration). All flat maps must be audited for org-scoping to prevent cross-tenant data leaks.

**Files:** New `src/coordinator/org-store.ts`, `src/coordinator/tenant-coordinator.ts`, `src/coordinator/registration.ts`. Major refactor of `src/coordinator/server.ts`, `src/coordinator/rest.ts`.
